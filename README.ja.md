## 📖 概要

> **このドキュメントは AI の支援を受けて作成されました。作者は内容の完全な正確性について責任を負いません。**

このプロジェクトは、Minecraft Bedrock Edition (統合版 / MCPE) 向けの高品質なスキンパックと、パッケージングおよび暗号化ツールを提供します。
**「スキンパックインジェクション (キャッシュ置換技術)」** を活用し、`skins.json`、`.lang`、およびスキン画像ファイルを変更・暗号化することで、公式マーケットプレイスのスキンパックをカスタムスキンにすり替えます。これにより、マルチプレイやサーバーでも正常に表示されるようになります。

⚠️ **免責事項**: 本プロジェクトおよびツールは技術交流および個人のカスタマイズ目的でのみ提供されています。商業目的での利用は禁止します。ゲームファイルの変更によるゲームのクラッシュやデータ損失について、作者は一切の責任を負いません。作業前に必ずバックアップを取ってください。

## 📁 ファイル構成

- `pack/` — コアリソースフォルダ。暗号化されていないカスタムファイル (`skins.json`、`geometry.json`、画像ファイル、`texts/` 言語ファイル) が含まれます。
- `ツールファイル` — プレーンテキストファイルを暗号化フォーマットに変換し、署名を生成するためのプログラム。
- `.gitignore` — 開発中の一次ファイル (`temp/` など) を除外するための設定。リリースには含まれません。

## ⚙️ インストールと置換手順

スキンパックをゲームに正常に読み込ませるために、以下の手順に厳密に従ってください。

### ステップ 1: 公式キャッシュフォルダの特定
Windows 11 のエクスプローラーを開き、以下のキャッシュパスに移動して「置換/上書き」したい公式スキンパックを探します (事前にマーケットプレイスでダウンロードしておく必要があります):
> **デフォルトパス:** 
> `C:\Users\あなたのユーザー名\AppData\Roaming\Minecraft Bedrock\premium_cache\skin_packs\`
> *(フォルダ名は通常 `u9bQ7FAljNM=` のような文字化けした文字列です)*

### ステップ 2: ターゲットフォルダの選択とクリーンアップ (⚠️ 非常に重要)
1. **ターゲットの選択**: 置換成功後にゲーム内の更衣室 (Dressing Room) で見つけやすくするために、識別しやすい公式スキンパック (例: **"1st Birthday Skin Pack"**) を選ぶことをお勧めします。
2. **IDの保持**: 文字化けした公式フォルダに入ったら、**必ず公式の `manifest.json` を残してください** (これは公式パックの身分証明書であり、絶対に削除してはいけません)。
3. **コンテンツのクリア**: フォルダ内の `manifest.json` 以外の **公式ファイルをすべて削除** します。

### ステップ 3: カスタムスキンファイルの導入
本プロジェクトの `pack/` フォルダ内のすべてのファイル (カスタムスキン、`.lang`、`skins.json` など) をコピーし、先ほどクリーンアップした公式フォルダに貼り付けます。
*(この時点で、対象フォルダ内には公式の `manifest.json` と貼り付けたカスタムファイルのみが存在するはずです)*

### ステップ 4: 暗号化ツールを使用したインプレース処理
Bedrock Edition はファイルの暗号化検証を行うため、そのままゲームを起動しても読み込まれません。付属の暗号化ツールを使用して最終処理を行います:
1. プロジェクト内の暗号化ツール `encryptor.exe` を実行します。
2. ツールのプロンプトに従い、**【ステップ 2】の文字化けした公式フォルダのパス** を貼り付けます。
3. ツールがフォルダ内のすべてのファイルの暗号化処理を自動的に行い、必要なデジタル署名 (`signatures.json` など) を生成します。

### ステップ 5: ゲームの再起動
Minecraft を起動し、更衣室に入ります。元の公式スキンパックが、本プロジェクトのカスタムスキンとカスタム名 (`.lang` ファイルで制御) に置き換わって表示されるはずです！

## 🛠️ 貢献・スキンの追加方法

MODの仕組みに詳しい方の PR (Pull Request) を歓迎します:
1. `pack/skins.json` にスキンの設定配列を追加し、対応する画像ファイルを `pack/` 内の適切なパスに配置します。
2. `pack/texts/` 内の言語ファイル (`.lang`) を編集し、スキンの表示名を設定します。
3. 暗号化をテストし、ゲーム内で問題なく動作することを確認してから PR を送信してください。
*( `.gitignore` で除外されている一時ファイルや無関係な暗号化出力物を PR に含めないでください )*
